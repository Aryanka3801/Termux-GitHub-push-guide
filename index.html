Step 1 — Install Termux & Git
Get the right version and install Git
Download Termux from F-Droid only (f-droid.org). The Play Store version is outdated and broken.
1 Update packages & install Git
Run these after opening Termux for the first time.
terminal
pkg update && pkg upgrade pkg install git
Step 2 — Configure Git Profile
Set your name and email — done once, saved forever
Use the same email as your GitHub account. This info appears in your commits.
2 Set your identity
Replace with your actual name and GitHub email.
terminal
git config --global user.name "YourName" git config --global user.email "youremail@gmail.com"
✓ Verify config
terminal
git config --global --list
Step 3 — Generate SSH Key
Create your key pair — the password that works automatically
3 Generate the key
Press Enter 3 times when prompted (skip passphrase).
terminal
ssh-keygen -t ed25519 -C "youremail@gmail.com"
4 View your public key
Copy everything that appears — from ssh-ed25519 to your email.
terminal
cat ~/.ssh/id_ed25519.pub
In Termux: Long press the output → Select All → Copy. The key is one long line starting with ssh-ed25519
Step 4 — Add Key to GitHub
Link your Termux SSH key to your GitHub account
5 Add SSH key on GitHub website
1
Go to github.com and sign in
2
Tap your profile picture → Settings
3
Left sidebar → SSH and GPG keys
4
Click New SSH key
5
Title: Termux — then paste your key in the Key box
6
Click Add SSH key ✅
6 Test the connection
Type yes (full word) when asked about connecting.
terminal
ssh -T git@github.com
Success looks like: Hi YourUsername! You've successfully authenticated...
Step 5 — Setup Storage Access
Allow Termux to access your Android files
7 Grant storage permission
Run once — grant access when Android asks.
terminal
termux-setup-storage
8 Navigate to your project folder
Replace YourFolder with your actual folder name.
terminal
cd /storage/emulated/0/Download/YourFolder
9 Fix ownership error (if it appears)
Run this if you see "dubious ownership" error.
terminal
git config --global --add safe.directory /storage/emulated/0/Download/YourFolder
Step 6 — Create Repo & Push
Create a GitHub repo and push your project files
10 Create repo on GitHub
1
Go to github.com → tap + → New Repository
2
Give it a name (e.g. my-android-app)
3
❌ Do NOT check "Initialize with README"
4
Click Create Repository
5
Copy the SSH URL → looks like git@github.com:Username/repo.git
11 First time push — run all commands
Replace Username and repo-name with yours.
first push — all commands
git init git add . git commit -m "first commit" git branch -M main git remote add origin git@github.com:Username/repo-name.git git push -u origin main
Every future update — just 3 commands
update commands
git add . git commit -m "describe your changes" git push
Push to a different repository
change remote
git remote remove origin git remote add origin git@github.com:Username/new-repo.git git push -u origin main
Quick Cheat Sheet
All important commands in one place
Command What it does
git status See changed files
git add . Stage all changes
git commit -m "msg" Save a snapshot
git push Upload to GitHub
git pull Download latest from GitHub
git log --oneline See commit history
git reset Undo last git add
git remote -v See current remote URL
git remote remove origin Remove current remote
ssh -T git@github.com Test SSH connection
cat ~/.ssh/id_ed25519.pub Show your SSH public key
