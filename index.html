Step 1 — Install Termux & Git
Get the right version and install Git
Download Termux from F-Droid only (f-droid.org). The Play Store version is outdated and broken.
1 Update packages & install Git
Run these after opening Termux for the first time.
terminal
pkg update && pkg upgrade pkg install git
Step 2 — Configure Git Profile
Set your name and email — done once, saved forever
Use the same email as your GitHub account. This info appears in your commits.
2 Set your identity
Replace with your actual name and GitHub email.
terminal
git config --global user.name "YourName" git config --global user.email "youremail@gmail.com"
✓ Verify config
terminal
git config --global --list
Step 3 — Generate SSH Key
Create your key pair — the password that works automatically
3 Generate the key
Press Enter 3 times when prompted (skip passphrase).
terminal
ssh-keygen -t ed25519 -C "youremail@gmail.com"
4 View your public key
Copy everything that appears — from ssh-ed25519 to your email.
terminal
cat ~/.ssh/id_ed25519.pub
In Termux: Long press the output → Select All → Copy. The key is one long line starting with ssh-ed25519
Step 4 — Add Key to GitHub
Link your Termux SSH key to your GitHub account
5 Add SSH key on GitHub website
1
Go to github.com and sign in
2
Tap your profile picture → Settings
3
Left sidebar → SSH and GPG keys
4
Click New SSH key
5
Title: Termux — then paste your key in the Key box
6
Click Add SSH key ✅
6 Test the connection
Type yes (full word) when asked about connecting.
terminal
ssh -T git@github.com
Success looks like: Hi YourUsername! You've successfully authenticated...
Step 5 — Setup Storage Access
Allow Termux to access your Android files
7 Grant storage permission
Run once — grant access when Android asks.
terminal
termux-setup-storage
8 Navigate to your project folder
Replace YourFolder with your actual folder name.
terminal
cd /storage/emulated/0/Download/YourFolder
9 Fix ownership error (if it appears)
Run this if you see "dubious ownership" error.
terminal
git config --global --add safe.directory /storage/emulated/0/Download/YourFolder
Step 6 — Create Repo & Push
Create a GitHub repo and push your project files
10 Create repo on GitHub
1
Go to github.com → tap + → New Repository
2
Give it a name (e.g. my-android-app)
3
❌ Do NOT check "Initialize with README"
4
Click Create Repository
5
Copy the SSH URL → looks like git@github.com:Username/repo.git
11 First time push — run all commands
Replace Username and repo-name with yours.
first push — all commands
git init git add . git commit -m "first commit" git branch -M main git remote add origin git@github.com:Username/repo-name.git git push -u origin main
Every future update — just 3 commands
update commands
git add . git commit -m "describe your changes" git push
Push to a different repository
change remote
git remote remove origin git remote add origin git@github.com:Username/new-repo.git git push -u origin main
GitHub Actions — Auto APK Build
Build your Android APK automatically on every push — no PC needed
GitHub Actions lets you build your APK in the cloud for free. Every time you git push from Termux, GitHub compiles your app automatically.
1 Create the workflow folder in Termux
Navigate to your project root, then create the required GitHub Actions directory.
terminal
cd /storage/emulated/0/Download/YourFolder mkdir -p .github/workflows
2 Install a text editor (nano)
You need a text editor inside Termux to create the YAML workflow file.
terminal
pkg install nano
3 Create the workflow YAML file
Create and open the workflow file with nano.
terminal
nano .github/workflows/build-apk.yml
4 Paste this workflow content
Copy everything below and paste it into nano. Then press Ctrl+O → Enter → Ctrl+X to save and exit.
build-apk.yml
# .github/workflows/build-apk.yml name: Build APK on: push: branches: [ main ] pull_request: branches: [ main ] jobs: build: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v3 - name: Set up JDK 17 uses: actions/setup-java@v3 with: java-version: '17' distribution: 'temurin' - name: Grant Gradle execute permission run: chmod +x gradlew - name: Build Debug APK run: ./gradlew assembleDebug - name: Upload APK artifact uses: actions/upload-artifact@v3 with: name: debug-apk path: app/build/outputs/apk/debug/app-debug.apk
5 Commit and push the workflow
Push the workflow file to GitHub — this will trigger your first automatic build.
terminal
git add .github/ git commit -m "Add GitHub Actions APK build workflow" git push
6 Download your built APK from GitHub
1
Go to your repo on github.com
2
Tap the Actions tab at the top
3
Click the latest Build APK workflow run
4
Scroll down to Artifacts → tap debug-apk to download
5
Unzip the downloaded file — your app-debug.apk is inside ✅
Build a Release APK instead
Replace the Build Debug APK step with this to build a release (unsigned) APK.
release variant
- name: Build Release APK run: ./gradlew assembleRelease - name: Upload Release APK uses: actions/upload-artifact@v3 with: name: release-apk path: app/build/outputs/apk/release/app-release-unsigned.apk
Release APKs built this way are unsigned. You need to sign them before publishing to the Play Store.
Common Build Errors & Fixes
!
gradlew: Permission denied → Make sure the chmod +x gradlew step is in your YAML
!
SDK not found → Your local.properties file is device-specific. Add it to .gitignore
!
Build fails on Java version → Change java-version: '17' to '11' for older projects
!
No artifacts uploaded → Check the APK path matches your actual module name (may not be app/)
Add local.properties to your .gitignore:
terminal
echo "local.properties" >> .gitignore git add .gitignore git commit -m "Ignore local.properties" git push
